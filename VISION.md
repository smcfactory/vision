### A startup factory that builds real products before launching tokens.

**Not another launchpad.** An AI-powered pipeline from idea to running business.

SMCFactory provides a new flow structure with features that benefit both founders who build and participants in the trenches. Every project that enters the pipeline goes through AI-powered evaluation before it launches. Founders receive real support — from idea validation through product development to launch execution. Participants receive a quality signal that doesn't exist anywhere else in the market: a factory-endorsed project with an evaluated idea, verified feasibility, and working code behind it.

> **SMCFactory proved the model by being its own first customer.** The factory identifies, evaluates, builds, and launches its own projects through the full pipeline. We don't ask founders and participants to trust a pipeline we haven't run ourselves. The only way to prove a Zero Human Company works is to be one first.

> **Why zero humans is the point, not the risk.** Traditional startups fail because humans get tired, lose interest, run out of money, or move on to the next thing. The project dies not because the idea was bad but because the people behind it quit. A Zero Human Company removes that failure mode entirely. The agents don't burn out, don't pivot to something shinier, don't stop showing up. The pipeline runs because it's built to run — not because someone is motivated enough to keep it going. Accountability doesn't come from a face on a website. It comes from code that's open, decisions that are logged on-chain, and a factory that already bet on itself first.

## What SMCFactory Does

**A full-cycle pipeline from idea to running business.**

SMCFactory is not a launchpad. It is a complete startup pipeline powered by an agentic swarm — dedicated AI agents that handle evaluation, development, launch execution, and post-launch operations. Every stage has a purpose. Every project earns its place.

<br>

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│             │     │             │     │             │     │             │     │             │
│  EVALUATE   │────▶│    BUILD    │────▶│   AMPLIFY   │────▶│   LAUNCH    │────▶│     RUN     │
│             │     │             │     │             │     │             │     │             │
│  Go/No-Go   │     │ Agent Swarm │     │ Media Stage │     │ Partner     │     │  Operate    │
│  Decision   │     │ Code & Audit│     │ Present     │     │ Infra       │     │  & Grow     │
│             │     │             │     │ Publicly    │     │             │     │             │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

<br>

**EVALUATE** — Dedicated AI agents review the idea — checking for genuine market demand, asymmetric opportunity, and initial technical feasibility. The output is a go or no-go decision. Projects that don't pass don't enter the pipeline. This single gate is what separates the factory from every launchpad that lets anything through.

**BUILD** — The agentic swarm handles development — writing code, testing, debugging, auditing smart contracts, and deploying. Task-specific agents work in parallel, shipping fast with strong test coverage.

**AMPLIFY** — Projects that pass evaluation and reach a working state enter a media stage. The founder presents the project publicly, capturing attention from the investment community before any token exists.

**LAUNCH** — Token creation happens through partnering infrastructure at early stages, aligned with the project's needs and maturity.

**RUN** — After launch, the factory continues operating the project — maintaining code, managing liquidity, responding to market signals. Projects are not abandoned at birth.

<br>

> **If an idea doesn't pass evaluation, it doesn't enter the pipeline. That filter doesn't exist anywhere else.**

## Two Operating Flows

**The factory builds for itself — and for founders.**

SMCFactory operates in two distinct modes. The internal flow is the factory functioning as a full Zero Human Company. The external flow is the factory serving as infrastructure for solo founders.

<br>

<table>
<tr>
<th width="50%">🟢 Internal Flow — Full ZHC</th>
<th width="50%">🟡 External Flow — Founder-Assisted</th>
</tr>
<tr>
<td>

**Factory Builds for Itself**

The factory identifies opportunities, evaluates them, builds the product, launches the token, and runs the business autonomously. No external founder involved. The agent swarm handles the complete cycle.

```
01 → Identify demand from market signals
02 → AI evaluation — go/no-go
03 → Agent swarm builds the product
04 → Launch through partner infrastructure
05 → Factory operates post-launch
```

</td>
<td>

**Factory Supports a Founder**

A solo founder brings an idea. The factory evaluates it — if the decision is go, the founder enters the pipeline. They can use factory capacities for building or build themselves. The factory provides the launch path.

```
01 → Founder submits idea
02 → AI evaluation — go/no-go endorsement
03 → Founder builds (optionally with factory)
04 → Media stage — founder presents publicly
05 → Launch through partner infrastructure
```

</td>
</tr>
</table>

<br>

The internal flow comes first. The factory proves the pipeline on its own projects before onboarding external founders. This is deliberate — the infrastructure must be battle-tested before others depend on it.
